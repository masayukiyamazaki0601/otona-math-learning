Chapter 02 / Unit 07

四則演算の基本法則（加法・減法）

Step 01

加法（足し算）の本質的意味と計算法則

足し算（加法）とは、複数の数量を「一つに合わせる（合併・追加）」操作です。数直線上では「右方向に進む」動作に対応します。

足し算をビジネスや日常生活で無意識に処理する際、私たちは以下の3つの重要な数理法則を使っています。これらを意識的に応用することで、暗算が劇的に効率化されます。

  1. 交換法則（Commutative Law）: $a + b = b + a$
    足す順番を入れ替えても、合計値は変化しません。例: $7 + 9$ が難しければ $9 + 7$ としても答えは同じ $16$ です。
  2. 結合法則（Associative Law）: $(a + b) + c = a + (b + c)$
    3つ以上の数を足す際、どこからグループにして足しても結果は同じです。例: $48 + 37 + 13$ を計算する際、先に後ろの $37 + 13 = 50$ を計算すると $48 + 50 = 98$ と暗算が容易になります。
  3. 単位元の存在（Additive Identity）: $a + 0 = a$
    「0」という数は、足し算において相手の数を変えない唯一の存在（単位元）です。
加法法則の視覚分析 (Visual Analysis of Addition Laws)

スライダーで A, B, C の値を変更し、足し算の順番を入れ替えても全体の長さ（合計値）が変わらないことを視覚的に確認しましょう。

数値 A 5
数値 B 8
数値 C 3
① 交換法則の視覚化 ($A + B$ vs $B + A$)
5 + 8 = 8 + 5 = 13
② 結合法則の視覚化 ( $(A + B) + C$ vs $A + (B + C)$ )
(5 + 8) + 3 = 5 + (8 + 3) = 16
Step 02

減法（引き算）の本質と注意点

引き算（減法）とは、ある全体量から一部を「取り去る（求残）」、または2つの数量の「差を比べる（求差）」操作です。数直線上では「左方向に進む」動作に対応します。

引き算を扱う際、最も重要かつ注意すべきルールは、「交換法則や結合法則が成り立たない」という点です。

  • 交換法則は不成立: $a - b \neq b - a$
    例: $10 - 3 = 7$ ですが、$3 - 10$ は $7$ にはなりません（のちの単元で学ぶ「負の数」になります）。順番を絶対に入れ替えてはいけません。
  • 結合法則は不成立: $(a - b) - c \neq a - (b - c)$
    例: $(10 - 5) - 2 = 3$ ですが、$10 - (5 - 2) = 7$ となり、計算結果が全く異なります。
Step 03

加法と減法の「相互逆演算」の関係

足し算と引き算は、表裏一体の関係にあります。数学的には「互いに逆の操作（逆演算）」と呼びます。

ある数に何かを足した後、同じ数を引くと元の数に戻ります。この関係こそが、計算の「検算（正しさの確認）」において最も強力な武器となります。特に実務の簿記や売上データの突き合わせ業務では、この関係を利用して入力ミスやシステムエラーを検出します。

$a + b = c \iff c - b = a$

Step 05

本単元のまとめ

  • 加法（足し算）の法則: 「交換法則」と「結合法則」が成り立ちます。これにより、計算の順序を自由に変えて暗算しやすく工夫することができます。
  • 減法（引き算）の制約: 交換法則・結合法則はどちらも成り立ちません。順序やカッコの位置を変えると、答えが変わってしまいます。
  • 逆演算の重要性: 加法と減法は互いに打ち消し合う逆関係（逆演算）です。これにより、$a + b = c$ の検証を $c - b = a$ で行うことができます。
