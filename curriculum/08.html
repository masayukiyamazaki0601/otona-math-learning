Chapter 02 / Unit 08

四則演算の基本法則（乗法・除法）

Step 01

乗法（掛け算）の意味と実務に活きる3つの法則

掛け算（乗法）とは、同じ数を繰り返し足し合わせる「累加」を効率的に行う計算です。また、縦と横の広がり（面積）や、何倍に拡大するかという「拡大（スケーリング）」の概念にも深く結びついています。

$a \times b = c$

掛け算の計算順序や展開を自在に操作するための3つの基本法則を学びましょう。

  1. 交換法則（Commutative Law）: $a \times b = b \times a$
    掛ける順番を前後させても、結果は同一です。例: $25 \times 7$ を計算する場合、頭の中で $7 \times 25$ に入れ替えると、$25$ の倍数（$25, 50, 75, 100 \dots$）のイメージから $175$ と素早く導くことができます。
  2. 結合法則（Associative Law）: $(a \times b) \times c = a \times (b \times c)$
    3つ以上の掛け算を行う際、どのペアから先に計算しても結果は同じです。例: $16 \times 5 \times 2$ を計算する際、後ろの $(5 \times 2) = 10$ を先に掛けることで、$16 \times 10 = 160$ と瞬時に暗算できます。
  3. 分配法則（Distributive Law）: $a \times (b + c) = a \times b + a \times c$
    カッコの中の足し算に対して、外側の掛け算をそれぞれに分配して計算できます。これは実務の暗算において極めて強力な武器になります。
    例: $7 \times 98$ を暗算する場合、$98$ を $(100 - 2)$ と分解します。 $$7 \times 98 = 7 \times (100 - 2) = 700 - 14 = 686$$ このように、複雑な掛け算をシンプルな「引き算（または足し算）」の組み合わせへと分解できます。

Step 02

除法（割り算）の本質と、2つの異なる視点

割り算（除法）とは、「全体を均等に分ける」操作です。割り算には、私たちが無意識に使い分けている2つの重要なアプローチが存在します。

  • ① 等分除（とうぶんじょ）: 「1つあたりの量」を求める
    全体を決められた数に等しく分ける操作です。例: 「12個のリンゴを3人で均等に分けると、1人あたり何個になるか？」（$12 \div 3 = 4$個）
  • ② 包含除（ほうがんじょ）: 「いくつ分に分かれるか」を求める
    全体の中に、特定の基準量がいくつ含まれているかを数える操作です。例: 「12個のリンゴを、1人3個ずつ配ると何人に配れるか？」（$12 \div 3 = 4$人）

また、割り算を扱うにあたって、数学上絶対に破ってはならない絶対ルールが1つあります。それが「0で割ることの禁止（Division by Zero）」です。

Step 03

なぜ「0で割ってはいけない」のか？（数理的証明）

「0で割ることは不可能」と学校で習いますが、その理由は引き算と掛け算の「逆演算の関係」から論理的に証明することができます。

割り算と掛け算は $a \div b = c \iff c \times b = a$ という関係にあります。ここで、$b = 0$（0で割る）としたときに何が起きるか、2つのケースに分けて考えます。

  • ケース①: $a$ が 0 以外のとき（例: $5 \div 0 = c$）
    逆演算の式に当てはめると、 $c \times 0 = 5$ となります。
    しかし、どんな数に0を掛けても結果は必ず0になるため、この式を満たす数値 $c$ はこの世に存在しません。数学ではこの状態を「不能（解なし）」と呼びます。
  • ケース②: $a$ が 0 のとき（例: $0 \div 0 = c$）
    逆演算の式に当てはめると、 $c \times 0 = 0$ となります。
    この式は、$c$ が $1$ であろうが $100$ であろうが、どんな数であっても成り立ってしまいます。答えが1つに決まらないため、数学ではこの状態を「不定（決定できない）」と呼びます。

このように、0以外の数で割るときは「不能」になり、0を0で割るときは「不定」になるため、一貫した計算結果を定義できません。これが、コンピュータのプログラムやExcelでも「0除算エラー（#DIV/0!）」が発生する理由です。

Step 05

本単元のまとめ

  • 乗法の法則: 交換法則・結合法則に加え、展開に便利な「分配法則」が成り立ちます。分配法則は、複雑な掛け算を簡単な暗算の組み合わせに分解する際に非常に役立ちます。
  • 除法の注意点: 順序の入れ替え（交換法則）は成り立ちません。また、数学上の定義崩壊を防ぐため、「0で割る」操作は完全に禁止されています。
  • 除法の2つの意味: 割り算には「1つあたりの量を求める（等分除）」と「いくつ分あるかを数える（包含除）」という2つのアプローチがあります。
