ホーム > コラム > 二次関数と最適価格設定

高校数学「二次関数」で決める！
最も儲かる「商品の価格設定」

ビジネスにおいて、最も頭を悩ませる意思決定の一つが**「値決め（価格設定）」**です。

価格を安くすれば、たくさん売れますが1個あたりの利益（マージン）は減ってしまいます。逆に、価格を高くすれば1個あたりの利益は増えますが、今度は売れる個数が減ってしまいます。 この**「高くして少なく売るか、安くして多く売るか」**というトレードオフのなかで、利益を最大化する「完璧な価格」はどうやって見つければよいのでしょうか。

実は、この難題に対する明確な答えを提示してくれるのが、高校で学ぶ**「二次関数」**と**「微分」**の知識です。

【体験】価格設定と利益最大化シミュレーター

まずは、パン屋さんをモデルにしたシミュレーターを動かしてみましょう。 原価（変動費）や固定費、そして「価格を上げるとどれくらい客足が遠のくか（価格感度）」を調整し、販売価格のスライダーを動かしてみてください。 販売価格の変化にともなって、需要と利益がどのように変化するか、リアルタイムに放物線が描かれます。

ステップ1：関係性を「一次関数」で表す

数学的に問題を解くためには、まず現実の事象を数式に置き換える（モデリングする）必要があります。 今回のパン屋さんにおいて、**価格と売れる個数（需要）の関係**を数式にしてみましょう。

市場調査によって、**「このパンは、価格を500円に設定したとき、月に200個売れる」**という基準が分かっているとします。 さらに、スライダーで設定した「価格感度」を b とします。例えば価格感度が 2.0 の場合、「価格が10円上がるごとに、需要が2個減る（＝1円上がるごとに0.2個減る）」という意味になります。

価格を P 円、そのときの需要量を Q 個と置くと、この関係は次の右下がりの**一次関数（直線）**で表されます。

Q = 200 - (b / 10) * (P - 500) = 200 - 0.2 * (P - 500) (価格感度 b=2.0 の場合) = 1200 - 0.2P

この式は、「もしタダ（価格 P = 0）で配るなら1200個の需要があるが、価格を1円上げるごとに需要が0.2個ずつ直線的に減っていく」という関係を示しています。 これが、シミュレーター右上にある**「需要曲線」**の正体です。

ステップ2：利益を「二次関数」へ展開する

次に、最終的に最大化したい目標である**「利益（放物線）」**の関数を組み立てます。 パン1個あたりの原価（変動費）を C 円、店舗の家賃などの固定費を F 円とします。

利益 y は、**「（販売価格 - 原価）× 販売個数 - 固定費」**という簡単な算数で計算できます。 ここに、先ほどの需要関数 Q = 1200 - 0.2P を代入してみましょう（原価 C = 300 円、固定費 F = 50,000 円とします）。

y = (P - C) * Q - F = (P - 300) * (1200 - 0.2P) - 50,000 = 1200P - 0.2P² - 360,000 + 60P - 50,000 = -0.2P² + 1260P - 410,000

価格 P の2次の項の係数が**マイナス（-0.2）**になっていることに注目してください。 これは、グラフを描いたときに**「上に凸（山型）の放物線」**になることを意味しています。 価格が安すぎても（原価割れや薄利多売の限界）、高すぎても（誰も買ってくれない）、利益はマイナス（赤字）になり、その中間の「どこか」に、山の一番高い場所（極大値）が存在することが、数理的に証明されたのです。

ステップ3：最適価格を求める（2つのアプローチ）

放物線の頂点、すなわち利益が最大となる価格 P を求めるには、数学的に2つの方法があります。

アプローチA：平方完成（高校数学の王道）

二次関数を y = a(P - p)² + q の形に変形し、頂点 (p, q) をあぶり出します。 先ほどの利益関数を変形してみましょう。

y = -0.2P² + 1260P - 410,000 = -0.2(P² - 6300P) - 410,000 = -0.2( (P - 3150)² - 3150² ) - 410,000 = -0.2(P - 3150)² + 0.2 * 9,922,500 - 410,000 = -0.2(P - 3150)² + 1,984,500 - 410,000 = -0.2(P - 3150)² + 1,574,500

この数式は、価格 P = 3150 円のときに二乗の項がゼロとなり、利益が最大値の**1,574,500円**になることを示しています（これは感度等の初期パラメータに基づいた例です）。

アプローチB：微分（実務における最強ツール）

平方完成は確実ですが、数式が複雑になると計算が非常に煩雑になります。そこで威力を発揮するのが**「微分」**です。 関数の頂点（極大値）では、**「接線の傾きがゼロ（0）」**になるという性質を利用します。利益関数 y を価格 P で微分して導関数を求め、それが 0 になる方程式を解きます。

y' = d/dP (-0.2P² + 1260P - 410,000) = -0.4P + 1260 接線の傾きが0となる条件は、y' = 0 であるため、 -0.4P + 1260 = 0 0.4P = 1260 P = 3150

平方完成に比べ、驚くほど簡単な計算で「最適価格は 3150 円である」という同じ答えが導き出せました。 実務での複雑な価格決定モデルや、競合の要素を入れた高次の関数を扱う際も、この「微分して0になる点を求める」というアプローチが基盤となっています。

まとめ：ビジネスを「どんぶり勘定」から「数理設計」へ

値決めを「勘」や「競合と同じ」に頼るのではなく、「価格変更に対する客足の変化（価格感度）」をテスト等で測定し、簡単な一次関数と二次関数にモデリングするだけで、理論上最も儲かる価格がピンポイントで導き出せます。 高校の授業で学んだ抽象的な「二次関数のグラフ」や「平方完成」は、現代の企業のマーケティング部門やデータサイエンティストが利益を設計するための、極めて強力な実践ツールなのです。
